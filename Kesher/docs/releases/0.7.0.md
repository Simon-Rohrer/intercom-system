# Kesher 0.7.0

Release date: 2026-03-31

## Highlights

- New Tauri-based native desktop application for macOS and Windows with integrated audio engine.
- Monorepo restructure using npm workspaces for shared client-core package across web and desktop.
- Optimized audio latency throughout the stack with runtime timing metrics and queue management.
- Enhanced CI/CD pipeline with native desktop builds for both ARM64 and AMD64 macOS architectures.
- Fixed 9 failing tests in client-core test suite to improve reliability.

## New Features

### Native Desktop Application (Tauri)

- Full desktop application for macOS (ARM64 + AMD64) and Windows built with Tauri.
- Integrated native audio engine in Rust for low-latency audio processing.
- Local desktop checks and configuration management for standalone operation.
- Desktop application can operate independently or bridge to a remote Kesher station server.

### Monorepo with npm Workspaces

- Restructured project layout: `packages/client-core/` contains shared code for both web and desktop.
- Web and desktop applications now consume the same `client-core` package via npm workspaces.
- Unified types, API layer, and common components across platforms.

### Performance & Latency Optimizations

- Reduced queue latency in desktop audio engine with incremental sync and runtime timing metrics.
- Tuned audio receiver/microphone for low-latency operation (44100 Hz, frame sizes optimized for network conditions).
- Room matrix routing stability improvements for reduced audio switching delays.
- Companion audio path latency reduction.

### macOS Desktop CI/CD

- GitHub Actions workflow split into separate ARM64 and AMD64 build jobs.
- Fixed Opus encoder dependency for macOS builds with proper CMAKE policy configuration.
- Artifact upload from target-specific `.dmg` paths for both architectures.

## Improvements

- Client-core test suite expanded and stabilized with 9 previously failing tests now passing.
- Desktop build and CI test targets added to Makefile for local development.
- GitHub Actions workflows improved for reproducible desktop builds.
- Page navigation support extended (page_home, page_jump) via companion integration.
- Companion paging stability improved with dynamic toggle support.

## Notable Fixes

- `packages/client-core/`: 9 failing tests resolved; test infrastructure solidified.
- `desktop/src-tauri/src/audio_engine.rs`: Native audio engine now correctly handles latency-critical paths.
- `github/workflows/`: Desktop build steps corrected for macOS runners (Opus, architecture targeting, artifact paths).
- `Makefile`: Added `desktop-build` and `desktop-test` targets for consistent local workflows.

## Architecture Notes

- **Desktop Application**: Built with Tauri, ships native Rust audio engine, operates as a companion source or standalone bridge.
- **Shared Package**: `packages/client-core/` is now the canonical source for API, types, and React components shared by web and desktop.
- **Audio Stack**: Unified low-latency tuning across web (browser WebAudio) and desktop (native Rust engine).
- **CI/CD**: Platform-specific builds ensure optimized binaries for macOS (Universal binary or separate ARM64/AMD64) and Windows.

## Configuration

No new configuration keys required. Desktop app inherits server URL and credentials from CLI or environment variables.

## Migration / Upgrade Notes

- No database migration required.
- Existing web deployments continue to work as-is.
- Desktop application is optional; web SPA remains the primary operator interface.
- npm workspace structure is internal; no changes needed for end users.

## Known Limitations

- Desktop app macOS build requires macOS 10.13+ for Tauri support.
- Audio bridging between web and desktop requires explicit server URL configuration.
- Windows .msi installer not yet available; .exe build process being finalized.
