# Kesher 0.5.2

Release date: 2026-03-27

## Highlights

- Fixed role-scoped Stream Deck image feedback routing behavior.
- Prevented deleted default roles/rooms from being re-seeded on backend restart.
- Added regression coverage for store seeding behavior and image stream handling.

## Improvements

- Store bootstrap now seeds default data only on first database creation.
- Bootstrap settings (including admin PIN) are ensured independently from demo seed data.
- Runtime backend logs are ignored via `.gitignore` to keep release workflows clean.

## Notable Fixes

- `backend/internal/app/store.go`: no unconditional reseed on reopen.
- `backend/internal/app/store_test.go`: added regression test for deleted default data persistence.
- `backend/internal/app/image_stream.go`: adjusted image feedback routing path.
- `backend/internal/app/image_stream_test.go`: expanded coverage for routing behavior.

## Migration / Upgrade Notes

- Existing databases will keep user deletions of default roles/rooms after restart.
- No configuration migration required for this patch release.
