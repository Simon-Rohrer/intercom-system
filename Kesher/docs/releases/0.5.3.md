# Kesher 0.5.3

Release date: 2026-03-28

## Highlights

- New incoming-call indicator button for Companion/Stream Deck with a persistent blink latch that stays active until explicitly acknowledged.
- Full acknowledgement flow: operators can confirm an incoming call from the intercom station or directly via the companion source button.
- Companion image effects are now driven by a configurable `image-effect-map.json`, making blink and flash behaviours declarative and easy to customise.
- Reply-to-caller blink is now correctly restricted to direct calls, avoiding spurious blink on room or broadcast PTT.

## New Features

### Incoming-call indicator & acknowledgement (Companion / Stream Deck)

- A dedicated "incoming call" button type is rendered on Stream Deck/Companion surfaces when a direct call arrives at the operator's role.
- The blink latch keeps the button flashing until the call is acknowledged — it survives hold-release cycles so operators always notice a missed call.
- Acknowledgement can be triggered from the web station view or by pressing the matching source button on any connected Stream Deck/Companion surface.
- Backend routes the acknowledgement event through the hub and clears the latch on all connected surfaces for that role.

### Image effect map (`image-effect-map.json`)

- A new top-level `image-effect-map.json` file (loaded via `config.yaml` `imageEffectMap` key) declares which image-stream button types trigger which visual effect (blink, flash, solid colour, etc.).
- The map is hot-reloadable and applies to both the WebSocket image bridge and the hardware Stream Deck feedback path.
- A default map is shipped with the repository; operators can override it by pointing `imageEffectMap` at a custom file.

## Improvements

- Reply-to-caller blink signal is now only emitted for `direct` call events; room-PTT and broadcast-PTT no longer trigger an unwanted blink on the reply button.
- Companion acknowledgement coverage expanded: hub routing and signal/reply metadata are updated atomically so repeated press-and-release sequences cannot leave stale latch state.

## Notable Fixes

- `backend/internal/app/hub.go`: blink-latch state is properly cleared on acknowledgement; reply-signal fan-out restricted to direct-call context.
- `backend/internal/app/server.go`: new HTTP/WS handlers for `incoming_call_ack` and image-effect-map endpoint; companion routing hardened against races.
- `backend/internal/app/image_stream.go`: effect-map lookup integrated into the per-button render path.
- `web/src/lib/streamDeckLabels.ts` / `streamDeckBridge.ts`: incoming-call button type rendered and ACK command wired up.

## Configuration

Add the following key to `config.yaml` to point at a custom effect map (optional — the bundled default is used when omitted):

```yaml
imageEffectMap: "image-effect-map.json"
```

## Migration / Upgrade Notes

- No database migration required.
- The `image-effect-map.json` file is bundled next to the binary; Docker images include it automatically via the updated `Dockerfile`.
- Existing Companion module installations will receive the incoming-call button type automatically after updating the backend; no module reconfiguration needed.
